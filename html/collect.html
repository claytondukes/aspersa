The collect tool
Aspersa Manual › The collect tool

The collect tool gathers a variety of information about a system for a short time. It is typically executed when the stalk tool detects a condition and wants to collect information to assist in diagnosis.

Command-Line Options and Environment Variables

This tool does not have any command-line options, but it does have configuration options embedded in the top of the code, so it can be customized.

How it Works

This tool first ensures that the disk where it's storing its data is not getting too full. If there is not enough free space, it exits with an error message. Barring that, it creates a lock to ensure that only one instance runs at a time, and then saves a variety of performance and status data into files in the configured directory. Files are named with a timestamp so they can be grouped together. The tool is MySQL-centric by default, and gathers quite a bit of diagnostic data that's useful for understanding the behavior of a MySQL database server.

Example Usage

To use this tool, simply configure it as desired and execute it with no options. Check the comments in the top of the source to see how to configure it.
